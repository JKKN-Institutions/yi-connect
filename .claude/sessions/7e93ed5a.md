### Lessons Learned
- **Routes 200 ≠ navigation works.** Agent Y verified 22/22 routes returned 200; missed that hrefs to those routes were broken. Three-layer-sweep: (1) route exists, (2) link prefix correct, (3) UI actually navigates. Captured as `feedback_routes_200_missed_link_layer.md`.
- **Middleware intercepts before layout.** When a Next.js redirect doesn't update despite verified deploy, search `middleware.ts` AND `lib/*/middleware.ts` FIRST. App-tree code (layouts/pages) can't override middleware. Captured as `feedback_middleware_intercepts_before_layout.md`.
- **Instrument before iterating.** When deploy N+1 fails to fix the bug, do NOT apply another code change. Encode a diagnostic param (e.g. `?_d=type-${session.type}`) into the suspect redirect — if the param appears, your code is the source; if not, hunt elsewhere. I burned 3 unnecessary deploys (~9 min) before doing this. Captured as `feedback_instrument_before_iterating.md`.
- **5-agent worktree fan-out scales linearly when ownership is non-overlapping at the file level.** ~110-bug previous session → today's 9-bug session. Pattern repeats well. Watch for over-literal scope wording ("ONLY that one line") — Agent C left 4 stragglers because of it. Better wording: "all hrefs in this file matching pattern X."
- **BUG-08 component-sharing assumption was wrong.** I told Agent E "this component is reused" — Agent E checked and found mentor/messages is a SEPARATE file. Cheap to verify, expensive to assume. Always check before claiming cross-consumer impact.

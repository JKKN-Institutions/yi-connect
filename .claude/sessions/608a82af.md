### Key Decisions
- **YiFi is an event, not a competition** — no jury/rounds/scoring. Organiser roles, not competition roles.
- **event_capabilities JSONB** — any event can enable routing/vows/dossiers. YiFi is just the first with all capabilities maxed. National events default all-on, chapter events default all-off.
- **Google-first auth eliminates access codes** — system finds you by email across all modules. Access codes are fallback only.
- **Don't replicate Stutzee** — build what makes it unnecessary. 7 Yi-specific differentiators that Stutzee can never build because it doesn't understand Yi.
- **PostgREST workaround** — yifi schema can't be in `db_schema` (causes PGRST002 cache build hang). Use SECURITY DEFINER plpgsql functions in public schema instead.

### Gotchas / Feedback Memories Saved
- Supabase CLI `--linked` connects to the linked project, not the one in .env.local. Always verify with `SELECT current_database()`.
- PostgREST PGRST002 = schema cache build hanging. Caused by adding complex schema to db_schema. Fix: use RPC functions in public schema with SECURITY DEFINER.

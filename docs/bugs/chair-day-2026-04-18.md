## Newly Discovered Bugs

### [BUG-CD-001] ALL Server Action mutations return HTTP 500 in production — CRITICAL
Every tested write path returns 500:
- POST `/events/new` (event create)
- POST `/communications/announcements/new` (send announcement)
- POST `/pathfinder/plans/new?vertical=…` (AAA plan create)
- POST `/pathfinder/health-card/new` (health card entry)
- POST `/events/{id}/checkin` (event checkin)

Reads all work (dashboard, event list, expenses, reimbursements, member-requests). Session is valid (verified via GET `/dashboard` after every failure).

Very likely causes to investigate:
- Server Action `serverActions.bodySizeLimit` / runtime mismatch in Next.js 16 build
- Missing Supabase service-role env var in Vercel production
- Missing `use cache` / `cacheLife` annotation post-upgrade
- RLS policy or enum change not applied to prod DB
- The commit 0393f6a (Merge Agent A) may have broken something universal

First step: pull latest Vercel production function logs for any of the 500 routes above — the stack trace will identify this in seconds. Every Chair-day action hits this wall.

### [BUG-CD-002] Public `/apply` returns HTTP 200 but silently drops the application
The `/apply` wizard completes all 4 tabs, submits cleanly (200 OK, no errors, no toast), but the record never appears in `/member-requests` → Pending tab (confirmed as Chair immediately afterwards). Either the server action is committing without inserting, or the insert is silently rolled back. This is WORSE than a 500 because the guest user thinks their application landed.

### [BUG-CD-003] Member has no RSVP/Register button on event detail
Logged in as `demo-member@yi-demo.com` and opened a PUBLISHED event (`/events/aaaa4d76-7385-425d-88d0-e413c6574813`). The page shows a chair/admin layout (QR Code button, RSVPs/Volunteers/Feedback/Docs tabs) and zero member-facing "Register" / "I'm going" button anywhere. WT-002 / QuickRSVP is either missing in production or hidden by role logic that thinks demo-member is an admin. `/m/events/{id}` returns 404, so no mobile fallback either.

### [BUG-CD-004] Check-in form display/value mismatch
On `/events/{id}/checkin` the Select shows "Self Check-in" as an option but its emitted value fails Zod with `Invalid option: expected one of "qr_code"|"manual"|"self_checkin"`. The display label uses a human-friendly variant while the enum expects the snake_case form — select options out of sync with schema.

### [BUG-CD-005] Silent submit failures across the app (no error toast)
Every 500 response leaves the user on the form with no toast, banner, or `[role=alert]` message. `react-hot-toast` is installed but not wired into the Server Action error path. Users cannot tell whether their click worked.

## What Still Blocks The Chair

1. **Nothing can be written.** This is a Saturday-morning show-stopper. Create event, publish, announce, plan AAA, log health card, check people in — all blocked by the same 500. Find the common root cause first.
2. **Members cannot self-register for events** — QuickRSVP is absent in prod for the demo-member account.
3. **Public application intake is silently broken** — worse than a visible error; users assume it worked.
4. **No user-visible error telemetry** — silent failures everywhere. Add a global Server Action error boundary that shows a destructive toast.

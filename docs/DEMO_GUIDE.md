# Yi Connect Demo Guide

This guide helps chapter leaders experience the Yi Connect platform from different role perspectives.

| `demo-cochair@yi-demo.com` | **Co-Chair** | Member management, events, stakeholders |
| `demo-ec@yi-demo.com` | **EC Member** | Event creation, view members, stakeholder access |

## How to Login

1. Go to **https://yi-connect.vercel.app/login**
2. Enter the demo email address
3. Click "Continue with Email"
4. Check inbox for magic link (check spam if needed)
5. Click the link to login

> **Note:** Demo accounts use magic link authentication (no password needed).

## Role Walkthrough

### As Chair (demo-chair@yi-demo.com)

**What you can do:**
1. View the chapter dashboard with all metrics
2. Access Settings to configure chapter options
3. Manage all members and their roles
4. Create and publish events
5. View financial summaries
6. Access full analytics
7. Send announcements to all members

**Key pages to explore:**
- `/dashboard` - Chapter overview
- `/members` - Member management
- `/events` - Event lifecycle management
- `/stakeholders` - School and college CRM
- `/settings` - Chapter configuration
- `/analytics` - Performance metrics

### As Co-Chair (demo-cochair@yi-demo.com)

**What you can do:**
1. Manage members (add, edit, view)
2. Create and manage events
3. View and update stakeholder relationships
4. Send communications to members
5. Approve certain actions

**Key pages to explore:**
- `/dashboard` - Chapter overview
- `/members` - Member management
- `/events` - Event management
- `/stakeholders` - CRM access
- `/communications` - Member communications

### As EC Member (demo-ec@yi-demo.com)

**What you can do:**
1. View member directory
2. Create events (draft mode)
3. View stakeholder information
4. Submit nominations for awards
5. View assigned vertical activities

**Key pages to explore:**
- `/dashboard` - Chapter overview
- `/members` - View members
- `/events` - Create draft events
- `/stakeholders` - View CRM
- `/verticals` - Vertical activities

## Demo Scenarios

### Scenario 1: Planning a MASOOM Session
1. Login as Chair or Co-Chair
2. Go to Stakeholders → Schools
3. Select "Government Higher Secondary School"
4. View past MASOOM sessions
5. Go to Events → Create New
6. Create a MASOOM event for that school

### Scenario 2: Tracking Yuva Program
1. Login as any role
2. Go to Stakeholders → Colleges
3. View "Demo Engineering College"
4. See past Yuva programs conducted
5. Check engagement level and next steps

### Scenario 3: Member Management
1. Login as Chair
2. Go to Members
3. View member list with filters
4. Click on a member to see profile
5. Check role assignments

### Scenario 4: Event Lifecycle
1. Login as EC Member
2. Create a draft event
3. Logout, login as Co-Chair
4. Review and approve the event
5. Logout, login as Chair
6. Publish the event

## Tips for Demos

1. **Show role differences**: Login as different roles to show permission levels
2. **Use sample data**: Verticals and stakeholders have realistic data
3. **Highlight mobile**: The app is fully responsive
4. **Show workflows**: Demonstrate event creation → approval → completion

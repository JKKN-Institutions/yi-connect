### Lessons Learned

- **Don't downgrade locked decisions.** User locked monorepo (Q2) via AskUserQuestion; I quietly downgraded to "3 apps + SSO" during reconnaissance. User caught it 6h later. Saved as `feedback_dont_downgrade_locked_decisions`.
- **Default schema routing dissolved the 8,777-call-site problem in 4 lines of code.** Setting `db: { schema: 'yi_connect' }` on the Supabase client meant zero `.from()` rewrites were needed. Captured in `project_yi_connect_unified_db`.
- **Parallel worktree agents reliably fail at high tool counts.** 3 of 9 hit API 500/529 across two swarm waves. Salvage rate was high — agents typically wrote files before crashing. Plan for this.
- **macOS sed `\s` does NOT match whitespace.** Use literal spaces or `[[:space:]]` throughout.
- **GENERATED ALWAYS columns with `array_to_string()` or `to_tsvector('english',...)` fail IMMUTABLE check.** Strip GENERATED, replace with plain column; re-add via trigger if needed.
- **PostgreSQL function overload with default arg creates ambiguity.** `func(uuid DEFAULT NULL)` + `func()` → "function not unique" on calls. Drop the no-arg overload.
- **Compatibility VIEW pattern beats mass code patches.** When source schema's column shape doesn't match destination's, create a VIEW with computed columns. Avoid touching 8,777 query sites.

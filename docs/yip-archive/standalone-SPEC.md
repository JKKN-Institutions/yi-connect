# YIP 2.0 Platform — Specification

> Generated by SDD workflow on 2026-03-31

## Overview

**Young Indians Parliament (YIP) Platform** — a web application for conducting mock parliament sessions for school students (Classes 9-12) across India. It serves Yi chapter organizers, jury members, and student participants at three competition levels: Chapter, Regional, and National. The platform replaces manual spreadsheets, paper scoring sheets, and mental tracking with a digital system that handles student allocation, live event management, jury scoring, and results.

---

## Functional Specification

### User Roles

| Role | Can Do | Cannot Do |
|------|--------|-----------|
| **Chapter Organizer** | Create events, import students, run allocation, manage live event, add jury, view/publish results | Access other chapter's events, modify locked scores |
| **Jury** | Score participants during live event, edit own draft scores | Score after organizer locks, see other jury scores before publish, access organizer controls |
| **Participant (Student)** | View own role/party/constituency, view schedule, see published results | See scores before publish, access other events, modify any data |

[ASSUMPTION]: No "National Admin" role in MVP. A single organizer dashboard serves all levels. National admin features deferred to Phase 2.

### Screens / Pages

#### S1: Landing Page
- **URL:** `/`
- **Who sees it:** Everyone (public)
- **What it shows:** YIP branding (Yi / Thalir / Bharat Rising / CII logos), brief description, two CTAs: "Organize an Event" (→ login) and "Join with Access Code" (→ code entry)
- **Actions:** Navigate to login or join
- **Edge cases:** None significant

#### S2: Organizer Login
- **URL:** `/login`
- **Who sees it:** Organizers
- **What it shows:** Email + password form
- **Actions:** Login via Supabase Auth
- **Edge cases:** Invalid credentials → error message. No signup in MVP — organizers are pre-created via Supabase dashboard or seed script.

#### S3: Access Code Entry
- **URL:** `/join`
- **Who sees it:** Students and Jury
- **What it shows:** Single input for 6-character access code
- **Actions:** Enter code → system looks up participant/jury by code → redirects to role-appropriate dashboard
- **Edge cases:**
  - Invalid code → "Code not found. Check your badge or contact organizer."
  - Event not live → "This event hasn't started yet. Check back on [date]."
  - Code already used on another device → allow (no device lock in MVP)

#### S4: Organizer Dashboard
- **URL:** `/dashboard`
- **Who sees it:** Organizers
- **What it shows:** List of their events with status badges (Draft, Registration Open, Live, Completed)
- **Actions:** Create new event, click event to manage it
- **Edge cases:** No events → show "Create your first YIP event" CTA

#### S5: Event Setup (3-step wizard)
- **URL:** `/dashboard/events/new` (and `/dashboard/events/[id]/setup`)
- **Who sees it:** Organizer of this event
- **What it shows:** Step 1: Basic info (name, level, chapter, city, state, dates, venue). Step 2: Central agenda and committee topics. Step 3: Registration settings.
- **Actions:** Fill form, save draft, move between steps
- **Edge cases:** Incomplete form → save as draft, allow return later

#### S6: Participant Management
- **URL:** `/dashboard/events/[id]/participants`
- **Who sees it:** Organizer
- **What it shows:** Table of registered students with columns: Name, School, Class, Party, Role, Constituency, Committee, Access Code. Search and filter.
- **Actions:**
  - Add student manually (form)
  - Import CSV (columns: name, school, class, phone, email, city, state)
  - Delete student (before allocation lock)
  - Export list as CSV
  - Generate access codes (auto-generated on import)
- **Edge cases:**
  - CSV with missing columns → show validation errors per row
  - Duplicate student (same name + school) → warn but allow
  - CSV encoding issues → try UTF-8, fallback to show raw

#### S7: Smart Allocation
- **URL:** `/dashboard/events/[id]/allocation`
- **Who sees it:** Organizer
- **What it shows:**
  - Before allocation: "Run Allocation" button with settings (number of committees, constituency exclusion toggle)
  - After allocation: Preview showing Party breakdown, Role assignments, Committee distribution, Constituency map
  - Each section is a table/grid the organizer can review
- **Actions:**
  - Run allocation → executes algorithm → shows preview
  - Manual override: change any student's party/role/committee/constituency via dropdown
  - Re-run allocation (clears previous, runs fresh)
  - Lock allocation → finalizes, generates access codes, enables live event
- **Edge cases:**
  - < 30 students → warn "Recommended minimum is 120 for full parliament experience"
  - Only 1 school → can't split ruling/opposition by school, use random 50/50 split
  - Re-run after manual overrides → warn "Manual changes will be lost"

**Allocation Algorithm:**
1. Group by school → school with most students = Ruling, 2nd = Opposition, rest alternating to balance
2. If only 1 school → random 50/50 split
3. Assign 6 speaker candidates (3 per side, prefer Class 11-12)
4. Assign PM (ruling, highest class), LoP (opposition, highest class)
5. Assign 8 Cabinet Ministers (ruling), 8 Shadow Ministers (opposition) — one per ministry
6. Assign remaining as MPs
7. Assign constituencies from master list, excluding student's home state
8. Distribute all participants across committees (balanced count, both sides represented)

#### S8: Jury Management
- **URL:** `/dashboard/events/[id]/jury`
- **Who sees it:** Organizer
- **What it shows:** Table of jury members with name and access code
- **Actions:** Add jury (name → auto-generate code), remove jury, copy code to clipboard
- **Edge cases:** No jury added → warn before going live

#### S9: Live Event Control Panel
- **URL:** `/dashboard/events/[id]/control`
- **Who sees it:** Organizer (the main moderator screen)
- **What it shows:**
  - Large current agenda item title
  - Countdown timer (big, centered)
  - Current speaker name + role + party (when in speeches)
  - Speaker queue (upcoming speakers)
  - Quick stats: participants checked in, scores submitted
  - Agenda sidebar: list of all items with status indicators
- **Actions:**
  - Start event (set status to Day 1 Live)
  - Advance to next agenda item
  - Start timer (90s default, configurable)
  - Pause/resume/reset timer
  - Next speaker / Skip speaker
  - Mark speaker complete
  - End day 1 / Start day 2 / Complete event
- **Edge cases:**
  - Browser refresh → state persists (all state in database)
  - Two organizers open control panel → both see real-time state, last write wins
  - Timer reaches 0 → visual/audio alert, does NOT auto-advance

#### S10: Projector View
- **URL:** `/event/[id]/display`
- **Who sees it:** Everyone in the room (projected on screen), no auth required
- **What it shows:** Read-only large-format display:
  - Current agenda item
  - Timer countdown (huge)
  - Current speaker name, role, party, constituency
  - YIP branding
- **Actions:** None (read-only)
- **Edge cases:** No current speaker → show agenda item only. Event not live → show "Event starts on [date]"

#### S11: Jury Scoring (Mobile)
- **URL:** `/jury` (after access code login)
- **Who sees it:** Jury members
- **What it shows:**
  - Current speaker info (synced from control panel via Realtime)
  - Scoring form: one slider per criterion (based on speaker's role rubric)
  - Running total out of 100
  - "Save Draft" and "Submit" buttons
  - History of scores submitted this session
- **Actions:**
  - Score current speaker (sliders for each criterion)
  - Save as draft (can edit later)
  - Submit score (final, but can edit until organizer locks)
  - View past scores and edit them
- **Edge cases:**
  - Jury opens scoring but no speaker is active → "Waiting for next speaker..."
  - Jury tries to score same person twice → update existing score
  - Poor internet → scores saved in localStorage, auto-submit when back online

**Scoring Rubrics (from YIP docs):**

| Role | Criterion 1 | Criterion 2 | Criterion 3 | Criterion 4 | Criterion 5 | Total |
|------|------------|------------|------------|------------|------------|-------|
| Speaker | Impartiality /30 | Leadership & Control /25 | Knowledge of Rules /20 | Clarity of Communication /15 | Time Management /10 | /100 |
| Deputy Speaker | Support to Speaker /30 | Impartiality /25 | Leadership /20 | Communication Skills /15 | Adaptability /10 | /100 |
| MP | Content & Substance /30 | Communication & Delivery /25 | Parliamentary Conduct /20 | Argumentation & Persuasion /15 | Teamwork & Collaboration /10 | /100 |

#### S12: Participant View
- **URL:** `/me` (after access code login)
- **Who sees it:** Students
- **What it shows:**
  - "Welcome, [Name]" with role badge
  - Your assignment: Party, Role, Ministry (if applicable), Constituency, Committee
  - Event schedule (Day 1 + Day 2 agenda)
  - If results published: Your scores and rank
- **Actions:** View only (no data entry in MVP)
- **Edge cases:** Results not published → "Results will be announced at the closing ceremony"

#### S13: Results Dashboard
- **URL:** `/dashboard/events/[id]/results`
- **Who sees it:** Organizer
- **What it shows:**
  - Overall leaderboard: all participants ranked by average jury score
  - Category rankings: Best Speaker, Best Deputy Speaker, Best MP (Ruling), Best MP (Opposition), Best Overall
  - Per-participant detail: individual jury scores, average, criteria breakdown
  - Jury consistency view: see if any jury is scoring unusually high/low
- **Actions:**
  - Compute results (aggregate scores across juries)
  - Override award if needed (manual)
  - Publish results (makes scores visible to students)
  - Export results as CSV
- **Edge cases:**
  - Not all participants scored → show warning with list of unscored
  - Only 1 jury → no averaging needed, flag as single-jury event
  - Tie → show both, organizer picks winner manually

#### S14: Scoring Progress (Organizer)
- **URL:** `/dashboard/events/[id]/scoring`
- **Who sees it:** Organizer
- **What it shows:**
  - Scoring progress: X of Y participants scored by each jury
  - Heat map: which participants have been scored, which haven't
  - Per-jury activity: when they last submitted a score
- **Actions:** Lock scores (prevent further edits), unlock (allow corrections)
- **Edge cases:** Jury hasn't scored anyone → highlight in red

### Data Model

| Entity | Key Fields | Relationships |
|--------|-----------|---------------|
| Season | name, year, is_active | has many Events |
| Event | name, level, status, dates, venue, chapter | belongs to Season, has many Participants, AgendaItems, JuryAssignments |
| Participant | name, school, class, access_code, party_side, role, ministry, constituency, committee | belongs to Event, has many Scores |
| AgendaItem | day, sequence, title, status, duration | belongs to Event, has many AgendaSpeakers |
| AgendaSpeaker | speaking_order, allotted_seconds, status | belongs to AgendaItem + Participant |
| JuryAssignment | jury_name, access_code | belongs to Event, has many Scores |
| ScoringRubric | name, target_role, criteria (JSONB), total_max | referenced by Scores |
| Score | criteria_scores (JSONB), total_score, status | belongs to JuryAssignment + Participant + Rubric |
| ScoreAuditLog | previous_scores, new_scores, changed_by | belongs to Score |
| Result | avg_score, rank, award_category | belongs to Event + Participant |
| Constituency | name, state | master data, referenced by Participants |

### Scope Boundaries

**In scope (MVP):**
- Event creation and setup
- CSV student import + manual add
- Smart allocation engine (party, role, constituency, committee)
- Access code authentication (students + jury)
- Live event control panel with timer and agenda state machine
- Projector display view
- Mobile jury scoring with rubric-based criteria
- Score audit trail
- Results computation and publishing
- Basic participant view (role card + schedule)

**Explicitly out of scope (MVP):**
- National Admin role and cross-chapter dashboard
- Question Hour digital submission and management
- Bill drafting workspace
- Digital voting (speaker election, bill voting)
- WhatsApp integration/notifications
- Student preparation materials
- Offline scoring (IndexedDB)
- Multi-level pipeline (Chapter → Regional → National promotion)
- Certificate generation
- Payment/registration fee collection
- Organizer checklist tracker
- Analytics and historical data

### Assumptions

- [ASSUMPTION]: Organizer accounts will be pre-created via Supabase dashboard or seed script. No self-signup for organizers in MVP.
- [ASSUMPTION]: Internet will be available at the venue (even if spotty). No full offline mode in MVP, but jury scoring will use localStorage as buffer.
- [ASSUMPTION]: Maximum 170 participants per event (per YIP guidelines). UI tables can handle this without pagination.
- [ASSUMPTION]: One organizer per event is sufficient for MVP. No multi-organizer collaboration features.
- [ASSUMPTION]: The 543 Lok Sabha constituencies will be seeded as master data for allocation.
- [ASSUMPTION]: Access codes are 6-character alphanumeric, unique per event. Printed on student badges by organizer.
- [ASSUMPTION]: Supabase free tier is sufficient for MVP (one project, < 500 MB database).

## Technical Plan

### Database Schema

See complete SQL schema in the plan research (21 tables total). Core tables:

| Table | Purpose | Key Columns |
|-------|---------|------------|
| `seasons` | Program year | name, year, is_active |
| `events` | Each YIP session | name, level, status, dates, current_agenda_item_id, live_timer_end |
| `participants` | Students | name, school, class, access_code, party_side, parliament_role, ministry, constituency_name, committee_name |
| `agenda_items` | Event schedule | day, sequence_order, title, status, duration_minutes |
| `agenda_speakers` | Speaking queue | participant_id, speaking_order, allotted_seconds, status |
| `scoring_rubrics` | Eval criteria per role | target_role, criteria (JSONB), total_max |
| `jury_assignments` | Jury access | event_id, jury_name, access_code |
| `scores` | Individual scores | jury_assignment_id, participant_id, criteria_scores (JSONB), total_score, status |
| `score_audit_log` | Change history | score_id, previous_scores, new_scores |
| `results` | Computed rankings | event_id, participant_id, avg_score, rank, award_category |
| `constituencies` | Master data (543) | name, state |

### Component Architecture

```
Key shared components:
- Timer (countdown from server timestamp, used in control panel + projector + jury)
- RoleBadge (colored badge showing PM/LoP/Minister/MP/Speaker)
- PartySideIndicator (Ruling = blue, Opposition = red)
- AgendaTimeline (vertical timeline of agenda items)
- ScoreSlider (0-N slider with label and description)
- DataTable (sortable/filterable table for participants, scores)
```

### Third-Party Services

| Service | Purpose | Config Needed |
|---------|---------|--------------|

### Key Technical Decisions

| Decision | Chosen | Alternative | Why |
|----------|--------|-------------|-----|
| Timer sync | Server UTC timestamp | Client-side countdown | All devices show same time, survives refresh |
| Scoring input | Sliders | Number inputs | Faster on mobile, less error-prone |
| Auth for students | Access codes | Email/password | Students are 14-17, no email, need instant access |
| Rubric storage | JSONB in rubrics table | Hardcoded columns | Flexible when Yi changes criteria |
| Real-time | Supabase Realtime (broadcast) | Polling | Low latency for timer/agenda sync |
| Participant data | Denormalized on participants table | Separate join tables | Fast reads during live event (no joins needed) |
